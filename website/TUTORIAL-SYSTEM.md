## üìö Tutorial Structure Example (Lab 1.1)

- **What**: Concept explanation

- **What**: Tool usage

- **Action**: Practice with tools

## üìù Creating Additional Tutorials

To create tutorials for other labs:

1. **Template**: Use `lab-tutorial.html` as base
2. **Update Header**: Module, lab number, title
3. **Create Steps**: 6-8 steps following the structure
4. **Add Content**: 
   - Explanations
   - Visual guides
   - Code examples
   - Practice exercises
5. **Knowledge Checks**: 3-5 questions per tutorial
6. **Link from Lab Page**: Update lab card to link to tutorial

## üéì Pedagogical Approach

The tutorial system follows these principles:

1. **I Do, We Do, You Do**: 
   - I Do: Instructor explains (Step 1-2)
   - We Do: Guided practice (Step 3-4)
   - You Do: Independent practice (Step 5-6)

2. **Zone of Proximal Development**:
   - Starts with what learners know
   - Builds to what they can learn with guidance
   - Ends with independent application

3. **Active Learning**:
   - Hands-on practice
   - Immediate application
   - Real-world scenarios

4. **Metacognition**:
   - Knowledge checks
   - Reflection prompts
   - Self-assessment

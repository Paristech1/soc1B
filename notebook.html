Module 1: Phishing Attack ‚Äî The Gateway Incident

Email header analysis, OSINT enrichment, PCAP analysis, and incident ticketing

Create a comprehensive video tutorial teaching Module 1: Phishing Attack ‚Äî The Gateway Incident from the SOC Analyst I course.

Learning Objectives: By the end of this tutorial, students should be able to extract and analyze email headers to identify spoofing, use OSINT tools to enrich IOCs with threat intelligence, analyze PCAP files for C2 and data exfiltration, and create professional incident tickets following NIST 800-61.

Topics to Cover:

  1. Phishing Triage Workflow: Explain the IR Checklist and Phishing Response Template. Show how phishing represents 80% of initial compromise vectors. Walk through the phishing kill chain: reconnaissance, weaponization, delivery, exploitation, and post-compromise activity.
  2. Email Header Analysis: Demonstrate how to extract headers from common email clients (Outlook, Gmail). Explain key fields: From, To, Subject, Date, Message-ID, Received, Return-Path, and X-Originating-IP. Show how to identify domain spoofing, verify sender domains against sending server IPs, and check SPF/DKIM/DMARC authentication results. Include a hands-on walkthrough analyzing a sample phishing email header.
  3. OSINT for Threat Enrichment: Teach how to use VirusTotal for file and domain reputation checks, AbuseIPDB for IP reputation, Whois for domain registration information, and Shodan for identifying exposed services. Demonstrate a complete OSINT investigation workflow on a suspicious domain, showing how to interpret threat intelligence reports and when to escalate based on severity.
  4. PCAP Analysis of Exfil Attempts: Show how to use Wireshark to analyze network traffic for post-compromise phishing activity. Demonstrate filtering for DNS queries, HTTP requests, and other protocols. Explain how to identify C2 connections, data exfiltration patterns, and connections to known malicious IPs or domains. Walk through analyzing a sample PCAP file from a phishing incident.
  5. NIST 800-61 Framework: Explain the four phases of incident response: preparation, detection and analysis, containment/eradication/recovery, and post-incident activities. Show how to create a professional incident ticket documenting all findings with evidence and timestamps.

Hands-On Labs to Reference:

  • Lab 1.1: Email Header Analysis and Spoofing Detection
  • Lab 1.2: OSINT Threat Enrichment
  • Lab 1.3: PCAP Analysis and Incident Ticket Creation

Teaching Style: Use visual demonstrations, step-by-step walkthroughs, real-world examples, and practical exercises. Include diagrams showing email header structure, the phishing kill chain, and the NIST 800-61 framework. Make it engaging and practical for visual learners.

Module 2: Unauthorized Access ‚Äî The Insider & External Threat

IAM abuse, privilege escalation, host log analysis, and cloud IAM investigation

Create a comprehensive video tutorial teaching Module 2: Unauthorized Access ‚Äî The Insider & External Threat from the SOC Analyst I course.

Learning Objectives: By the end of this tutorial, students should be able to investigate IAM abuse and privilege escalation attacks, analyze host logs (Linux and Windows) for unauthorized access, investigate cloud IAM abuse using AWS CloudTrail and GuardDuty, and detect lateral movement across systems.

Topics to Cover:

  1. IAM Abuse & Privilege Escalation: Explain AWS IAM policy investigation workflows. Show how to identify privilege escalation attempts, unauthorized permission changes, and IAM policy misconfigurations. Demonstrate Linux permissions and how attackers exploit them. Walk through investigating failed login attempts in both Linux auth.log and AWS CloudTrail logs.
  2. Host Log Analysis: Teach Linux log analysis using journalctl commands. Show Windows Event Log (Security) analysis for unauthorized access. Demonstrate how to create user activity timelines from host logs. Explain key log entries to look for: failed logins, privilege escalations, suspicious user activity, and file access patterns.
  3. Lateral Movement Detection: Explain common attacker tools and techniques for lateral movement. Show how to detect SSH brute force attacks in network traffic. Demonstrate analyzing network traffic for suspicious file transfers and unauthorized connections. Walk through identifying patterns that indicate lateral movement across systems.
  4. Cloud-Specific Unauthorized Access: Teach how to investigate AWS GuardDuty alerts related to unauthorized access. Show how to analyze S3 bucket misconfigurations and unauthorized access attempts. Demonstrate CloudTrail log analysis for IAM abuse, including identifying suspicious API calls, unusual access patterns, and privilege escalation attempts.

Hands-On Labs to Reference:

  • Lab 2.1: IAM Abuse & Privilege Escalation - Investigating Failed Login Attempts
  • Lab 2.2: Host Log Analysis & Timeline - User Activity Timeline Creation
  • Lab 2.3: Cloud IAM Investigation - Detecting SSH Brute Force and Suspicious File Transfers

Teaching Style: Use visual demonstrations of log analysis, step-by-step command walkthroughs, real-world attack scenarios, and practical exercises. Include diagrams showing IAM policy structures, log analysis workflows, and lateral movement attack chains. Make it engaging and practical for visual learners.

Show how to identify evidence of data exfiltration on Linux systems, including file timestamps, access patterns, and deleted file recovery.

Module 4: DDoS Attack ‚Äî Availability and Resilience

DDoS detection, botnet analysis, mitigation strategies, and cloud defenses

Create a comprehensive video tutorial teaching Module 4: DDoS Attack ‚Äî Availability and Resilience from the SOC Analyst I course.

Learning Objectives: By the end of this tutorial, students should be able to identify different DDoS attack types and signatures, create rate-based detection logic, analyze network forensics for DDoS attacks, and implement cloud mitigation strategies.

Topics to Cover:

  1. DDoS Attack Types & Signatures: Explain Layer 3/4 attacks (SYN Flood, UDP Flood) versus Layer 7 attacks (HTTP Flood). Show how to identify botnet traffic patterns in NetFlow data. Demonstrate the characteristics of different DDoS attack types and how they manifest in network traffic.
  2. Rate-Based Detection Logic: Teach threshold setting and alert tuning workflows. Show how to create simple rate-limit alerts in a SIEM-like environment. Explain how to balance false positives with detection sensitivity. Demonstrate creating detection rules for DDoS patterns.
  3. Network Forensics for DDoS: Show how to analyze high-volume PCAP files for DDoS attacks. Demonstrate identifying source IPs and attack patterns. Explain how to trace a DDoS attack back to its originating network. Walk through analyzing network traffic to identify attack signatures and patterns.
  4. Cloud Mitigation Strategies: Explain AWS Shield, CloudFront, and WAF configuration basics. Show how to implement DDoS protection in cloud environments. Demonstrate creating a step-by-step mitigation plan and workflow for responding to DDoS attacks.

Hands-On Labs to Reference:

  • Lab 4.1: DDoS Detection - Identifying Botnet Traffic Patterns in NetFlow Data
  • Lab 4.2: Rate-Based Detection - Creating a Simple Rate-Limit Alert

Teaching Style: Use visual demonstrations of network traffic patterns, step-by-step detection rule creation, real-world DDoS scenarios, and practical exercises. Include diagrams showing DDoS attack types, network traffic flows, and mitigation architectures. Make it engaging and practical for visual learners.

Module 5: Malware Attack ‚Äî The Hostile Takeover

Static/dynamic analysis, memory forensics, reverse engineering, and malware triage

Create a comprehensive video tutorial teaching Module 5: Malware Attack ‚Äî The Hostile Takeover from the SOC Analyst I course.

Learning Objectives: By the end of this tutorial, students should be able to perform malware triage, conduct static and dynamic analysis, use memory forensics to identify malicious processes, and understand reverse engineering basics.

Topics to Cover:

  1. Malware Triage Workflow: Explain the malware triage checklist and sandbox analysis workflow. Show how to perform static analysis including extracting strings, calculating hashes, and analyzing file metadata. Demonstrate initial malware assessment techniques and when to escalate for deeper analysis.
  2. Dynamic Analysis (Safe Environment): Introduce sandboxing and behavioral analysis concepts. Show how to observe file system changes, registry modifications, and network activity in a controlled environment. Walk through a dynamic analysis workflow demonstrating how malware behaves when executed.
  3. Memory Forensics: Introduce the Volatility Framework and key commands (pslist, connscan, netscan). Show how to identify malicious processes and network connections in a memory dump. Demonstrate analyzing memory dumps to find evidence of malware execution, including process injection, network connections, and file system artifacts.
  4. Reverse Engineering Basics: Introduce disassemblers like Ghidra and IDA Free. Show how to identify malware families and common patterns. Explain basic reverse engineering concepts and how they apply to malware analysis. Demonstrate creating a comprehensive malware triage summary report.

Hands-On Labs to Reference:

  • Lab 5.1: Static Analysis - Strings, Hashes, File Metadata
  • Lab 5.2: Dynamic Analysis - Observing File System/Registry Changes
  • Lab 5.3: Memory Forensics - Identifying Malicious Processes and Network Connections

Teaching Style: Use visual demonstrations of analysis tools, step-by-step analysis walkthroughs, real-world malware samples (in safe environments), and practical exercises. Include diagrams showing malware lifecycle, analysis workflows, and memory structures. Make it engaging and practical for visual learners. Emphasize safety and proper containment procedures.

Module 6: Detection Engineering ‚Äî Building the Defenses

Suricata rules, YARA rules, Sigma rules, and SIEM integration

Create a comprehensive video tutorial teaching Module 6: Detection Engineering ‚Äî Building the Defenses from the SOC Analyst I course.

Learning Objectives: By the end of this tutorial, students should be able to understand the detection lifecycle, write Suricata rules for network detection, create YARA rules for malware detection, and develop Sigma rules for SIEM integration.

Topics to Cover:

  1. Introduction to Detection Engineering: Explain the Detection Lifecycle: Identify, Develop, Test, Deploy, and Tune. Show how detection engineering fits into SOC operations. Demonstrate the importance of creating effective detection rules to catch threats early.
  2. Suricata Rule Creation: Teach Suricata rule syntax, including common keywords (content, flow, sid). Show how to write a basic Suricata rule (e.g., detecting a specific HTTP header). Demonstrate testing and tuning a Suricata rule against synthetic traffic. Explain how to reduce false positives and optimize rule performance.
  3. YARA Rule Creation: Explain YARA rule syntax, metadata, and string matching. Show how to write a YARA rule to detect a specific malware sample. Demonstrate using YARA for file-based detection and how to test rules against known samples. Explain condition logic and how to create effective malware signatures.
  4. Sigma Rule Creation: Introduce Sigma as a generic SIEM rule format. Show how to convert Sigma rules to SIEM queries. Demonstrate creating detection rules that can be used across different SIEM platforms. Explain how Sigma rules enable detection-as-code practices.

Hands-On Labs to Reference:

  • Lab 6.1: Writing a Basic Suricata Rule - Detecting a Specific HTTP Header
  • Lab 6.2: Testing and Tuning a Suricata Rule against Synthetic Traffic
  • Lab 6.3: Writing a YARA Rule to Detect a Specific Malware Sample

Teaching Style: Use visual demonstrations of rule syntax, step-by-step rule writing walkthroughs, real-world detection scenarios, and practical exercises. Include diagrams showing rule structures, detection logic flows, and SIEM integration patterns. Make it engaging and practical for visual learners. Show code examples and explain each component clearly.

Lab Generation Prompts

Copy and paste these prompts into Perplexity Labs to generate additional hands-on labs for each module section

How to Use These Prompts

  1. Select a prompt below that matches the module section you want to expand
  2. Click the "Copy Prompt" button to copy the entire prompt
  3. Paste it into Perplexity Labs
  4. Perplexity will generate a complete lab with step-by-step instructions, sample data, and deliverables
  5. Review and customize the generated lab as needed

Module 1: Phishing Attack

1.1 Phishing Fundamentals and Email Header Analysis

Create a comprehensive hands-on lab for SOC Analyst I students on "Phishing Fundamentals and Email Header Analysis". The lab should: - Be designed for beginners with basic Linux command-line knowledge - Use only open-source tools (no paid platforms) - Include step-by-step instructions that walk learners through each action - Provide synthetic email header data for analysis - Cover: extracting email headers from different email clients (Gmail, Outlook, Apple Mail), reading Received headers bottom-to-top, identifying key header fields (From, Return-Path, X-Originating-IP, Received-SPF), interpreting SPF/DKIM/DMARC results, and identifying spoofing indicators Deliverables should include: - A sample spoofed email header with clear indicators - Step-by-step analysis instructions - A checklist of spoofing indicators to identify - A one-page analysis document template - Knowledge check questions with answers Format the lab following instructional design best practices with clear learning objectives, prerequisites, real-world context, and progressive complexity.

1.2 OSINT Threat Enrichment

Create a hands-on lab for "OSINT Threat Enrichment" for SOC Analyst I students. The lab should: - Be designed for beginners - Use open-source OSINT tools: VirusTotal, AbuseIPDB, Whois, and Shodan - Include step-by-step walkthroughs for each tool - Provide sample IOCs (IP addresses, domains, file hashes) extracted from a phishing email - Cover: checking IOCs on VirusTotal and interpreting detection ratios, querying AbuseIPDB for IP reputation, using Whois to check domain registration details, searching Shodan for exposed services, and compiling findings into a threat intelligence report Deliverables should include: - Sample IOCs from a phishing investigation - Tool-specific instructions with screenshots/descriptions - A threat enrichment report template - A comparison table showing results from all tools - Best practices for OSINT analysis Include real-world context explaining why OSINT enrichment is critical for SOC analysts.

1.3 PCAP Analysis of Phishing Post-Compromise

Create a hands-on lab on "PCAP Analysis for Phishing Post-Compromise Investigation" for SOC Analyst I students. The lab should: - Use Wireshark (open-source) for PCAP analysis - Include a sample PCAP file with synthetic phishing compromise traffic - Cover: filtering HTTP traffic for malicious downloads, identifying C2 (command-and-control) connections, analyzing DNS queries for suspicious domains, detecting data exfiltration patterns, and creating a timeline of events Deliverables should include: - A sample PCAP file with embedded malicious activity - Wireshark filter cheat sheet for phishing investigations - Step-by-step analysis instructions - A timeline template for documenting findings - An incident ticket following NIST 800-61 framework The lab should demonstrate how email header analysis leads to network forensics when a user clicks a malicious link.

Module 2: Unauthorized Access

2.1 IAM Abuse and Privilege Escalation

Create a hands-on lab on "IAM Abuse and Privilege Escalation Detection" for SOC Analyst I students. The lab should: - Focus on AWS CloudTrail log analysis (open-source AWS CLI) - Include synthetic CloudTrail logs showing IAM abuse - Cover: identifying unauthorized IAM user creation, detecting privilege escalation attempts (AssumeRole abuse), analyzing policy changes, identifying suspicious API calls from external IPs, and creating an incident timeline Deliverables should include: - Sample CloudTrail logs with IAM abuse indicators - AWS CLI commands for log analysis - A workflow diagram for IAM investigation - An incident report template section for IAM abuse - Red flags checklist for IAM abuse detection Include both AWS cloud and on-premise IAM scenarios.

2.3 Cloud IAM Investigation
